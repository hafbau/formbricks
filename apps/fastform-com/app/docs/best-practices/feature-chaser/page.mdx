#### Best Practices

# Feature Chaser

Following up on specific features only makes sense with very targeted surveys. Fastform is built for that.

## Purpose

Product analytics never tell you why a feature is used - and why not. Following up on specific features with highly relevant questions is a great way to gather feedback and improve your product.

## Preview

<DemoPreview template="Feature Chaser" />

## Fastform Approach

- Trigger form at exactly the right point in the user journey
- Never ask twice, keep your data clean
- Prevent form fatigue with global waiting period

## Overview

To run the Feature Chaser form in your app you want to proceed as follows:

1. Create new Feature Chaser form at [app.fastform.com](http://app.fastform.com/)
2. Setup a user action to display form at the right point in time

### 3. Set up a trigger for the Feature Chaser form:

Before setting the right trigger, you need to identify a user action in your app which signals, that they have just used the feature you want to understand better. In most cases, it is clicking a specific button in your product.

You can create [Code Actions](/docs/actions/code) and [No Code Actions](/docs/actions/no-code) to follow users through your app. In this example, we will create a No Code Action.

There are two ways to track a button:

1. **Trigger by innerText:** You might have a button with a unique text at the end of your feature e.g. "Export Report". You can setup a user Action with the according `innerText` to trigger the form, like so:

<Image
  src={ActionText}

  quality="100"
  className="rounded-lg max-w-full sm:max-w-3xl"
/>

2. **Trigger by CSS Selector:** In case you have more than one button saying ‚ÄúExport Report‚Äù in your app and only want to display the form when one of them is clicked, you want to be more specific. The best way to do that is to give this button the HTML `id=‚Äúexport-report-featurename‚Äù` and set your user action up like so:
